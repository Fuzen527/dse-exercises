Merge Sort utilizes a technique called Divide-and-Conquer. First, the problem is divided into sub-problems (which are further divided into sub-sub-problems). After solving each of the sub-problems, their results are combined, thus solving the main problem.

The figure on the right shows the steps to sort an array of 6 integers $a[1..6]$ using merge sort. Steps to divide a problem into sub-problems are represented by red arrows. As you can see, the first step is to divide the problem of size 6 into two sub-problems of size 3. Then to sort the left sub-problem (sub-array $a[1..3]$), we can divide it into two sub-sub-problems of size 2 (sub-array $a[1..2]$) and 1 (sub-array $a[3..3]$) respectively. When the size is odd, whether the left or right sub-problem gets an extra element is entirely up to you. When the sub-problem size becomes 1, we cannot further divide the sub-problem. Also, a sub-array of size 1 is always sorted.

When both sub-problems has been conquered, we can combine the two sub-arrays using the algorithm in the previous exercise, illustrated by the blue arrows. But before we write the function which divides the problem into sub-problems, let's verify that our sub-array merging algorithm actually works.

You will sort an integer array of size $N$ in $M$ specified steps. In a step, you will be told to merge two already-sorted sub-arrays: $a[l_1 .. r_1]$ and $a[l_2 .. r_2]$, such as those in magenta in the figure. The two sub-arrays will be adjacent, i.e., $r_1 + 1 = l_2$. After merging the two sub-arrays, output the merged sub-array $a[l_1 .. r_2]$. The steps will be in a way such that the array can be sorted successfully if you sub-array merging algorithm is correct.

Input

The first line contains two integers $N$ and $M$, the size of the array and the number of steps. ($2 \le N \le 100$, $1 \le M \le 1000$)

The next line contains $N$ integers separated by spaces representing the array $a$. The integers are in $[-2147483648, 2147483647]$.

Each of the next $M$ lines contains 4 integers to describe a step: $l_1$, $r_1$, $l_2$ and $r_2$. This means to merge the sub-arrays $a[l_1 .. r_1]$ and $a[l_2 .. r_2]$ ($1 \le l_1 \le r_1 \lt l_2 \le r_2 \le N$, $r_1 + 1 = l_2$).

Output

After each step, output the sub-array $a[l_1 .. r_2]$ in a line with elements separated by spaces.
